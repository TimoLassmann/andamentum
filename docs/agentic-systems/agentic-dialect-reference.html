The Agentic Dialect

A minimal way to build agentic systems on a graph engine and an LLM client.

3 surfaces · 2 roles · 1 agent · 2 disciplines · 8 laws · 5 shapes

Few primitives, orthogonal, no hidden control flow, data and code kept apart. If a thing you are about to write is not obviously one of the pieces below, it is in the wrong place.

The laws
L1 · Surface placementreview
Given → Deps. Produced and read widely → State. Produced for the next step → Inputs.
L2 · Thin orchestrator, fat workerlint
Orchestrators route and dispatch; workers compute. A worker never imports the graph engine.
L3 · State written only by orchestratorsreview
Workers return values; the orchestrator assigns them. The join is the sole writer during parallelism.
L4 · Routing static, declarative, deterministictest
Every successor is known from the code; branch among declared edges, never synthesize one. No clock, randomness, or untrusted text.
L5 · Loops, recursion, fan-out are boundedlint
The bound — count or fan-out width — traces to a Deps value or a named constant.
L6 · The model is a component, not the controllerreview
Flow lives in the graph. An agent answers a question; it does not drive the pipeline.
L7 · Typed boundaries; fail loudtype-check
A structured schema on every edge — no untyped dict[str, Any]. A missing service raises.
L8 · Effects are idempotentreview
A world-changing worker must be safe to run more than once — loops, retries, and resume all re-enter it.
